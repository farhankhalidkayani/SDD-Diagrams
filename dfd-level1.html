Data Flow Diagram (Level 1)

This Level 1 Data Flow Diagram (DFD) breaks down the Go-Rent system into its major functional processes and shows how data flows between these processes.

It expands on the context diagram by detailing the main subsystems and their interactions.

Major Processes:

  • User Management: Handles user registration, authentication, and profile management
  • Search Engine: Processes search queries and returns relevant rental listings
  • Listing Management: Manages rental item listings, updates, and availability
  • Booking Management: Handles the creation and processing of bookings
  • Payment Processing: Manages payment transactions and security deposits
  • Notification Service: Sends notifications to users about bookings and payments
  • Review & Rating: Handles user reviews and ratings
  • Database Service: Stores and retrieves all system data

Key Data Flows:

  • User authentication data flows from User Management to Booking Management
  • Search queries flow from User Management to Search Engine
  • Listing data flows from Search Engine to Listing Management for display
  • Booking data flows from Booking Management to Payment Processing
  • Transaction data is stored in the Database
  • Booking data flows to Review & Rating for post-rental feedback
