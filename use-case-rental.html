Rental Process Use Case Diagram

This detailed use case diagram focuses on the rental process in the Go-Rent system, showing the various use cases and their relationships.

It demonstrates the use of include and generalization relationships between use cases as specified in standard UML notation.

UML Notation Legend

  • Actors: Represented as stick figures (Renter, Provider, Payment System)
  • Use Cases: Represented as ovals inside the system boundary
  • System Boundary: Rectangle containing all use cases
  • Include Relationship: Dashed arrow with <<include>> stereotype, pointing from base use case to included use case
  • Generalization Relationship: Solid line with hollow arrowhead, pointing from specialized use case to general use case
  • Associations: Solid lines connecting actors to use cases they interact with

Use Case Relationships:

Include Relationships

An include relationship specifies that a use case contains the behavior defined in another use case.

  • Book Item includes Verify Item Availability: When booking an item, the system must verify the item is available for the requested dates
  • Book Item includes Calculate Total Cost: When booking an item, the system calculates the total cost including rental fee and deposit
  • Make Payment includes Process Security Deposit: Payment processing includes handling the security deposit
  • Make Payment includes Generate Invoice: Creating an invoice is part of the payment process
  • Manage Booking includes Send Notifications: Booking management includes sending notifications about status changes
  • Accept/Reject Booking includes Update Item Status: When a provider accepts or rejects a booking, the item status must be updated
  • Return Item includes Refund Security Deposit: The return process includes refunding the security deposit if the item is in good condition

Generalization Relationships

Generalization shows that one use case is a specialized form of another.

  • Cancel Booking generalizes Manage Booking: Canceling a booking is a specialized form of booking management
  • Extend Rental Period generalizes Manage Booking: Extending a rental period is a specialized form of booking management

Key Use Cases Explained:

  • Search for Items: Renters search the platform for items they want to rent
  • Book Item: Renters request to book a specific item for specific dates
  • Make Payment: Renters pay for the booking through the platform
  • Accept/Reject Booking: Providers review booking requests and either accept or reject them
  • Return Item: Process for returning the item at the end of the rental period
  • Extend Rental Period: Renters can request to extend the duration of the rental
  • Review Experience: After completion of the rental, both parties can leave reviews
