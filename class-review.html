Review and Rating Class Diagram

This class diagram shows the structure of the Review and Rating component in the Go-Rent system.

It illustrates the class responsible for handling user reviews and ratings for rental items and providers.

        classDiagram
            class Review {
                +reviewId : int
                +itemId : int
                +reviewerId : int
                +providerId : int
                +bookingId : int
                +rating : int
                +comment : string
                +createdAt : datetime
                +status : enum
                +createReview(bookingId, rating, comment) : Review
                +getReviewById(reviewId) : Review
                +getItemReviews(itemId) : Review[]
                +getUserReviews(userId) : Review[]
                +calculateAverageRating(itemId) : decimal
                +moderateReview(reviewId, status) : boolean
                +reportReview(reviewId, reason) : boolean
            }

Key Class:

  • Review: Represents user reviews and ratings for items and providers

Key Methods:

  • createReview(): Creates a review for a completed booking
  • getItemReviews(): Retrieves all reviews for an item
  • getUserReviews(): Retrieves all reviews for a user
  • calculateAverageRating(): Calculates average rating for an item
  • moderateReview(): Updates review status for moderation
  • reportReview(): Allows users to report inappropriate reviews

Review Attributes:

  • reviewId: Unique identifier for the review
  • itemId: The item being reviewed
  • reviewerId: User who wrote the review
  • providerId: Provider of the rental item
  • bookingId: Associated booking that led to this review
  • rating: Numerical rating (typically 1-5)
  • comment: Textual feedback
  • status: Review status (pending, approved, rejected)
